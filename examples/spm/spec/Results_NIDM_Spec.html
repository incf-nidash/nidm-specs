In neuroimaging, the most common approach to analyse and draw conclusion on the data is to analyse each voxel (volume element) in parallel. This type of approach is sometimes referred as "massively univariate" because each voxel is analysed using a univariate test. This type of analysis can be applied to a large range of datasets including: functional MRI, anatomical MRI (with Voxel-Based Morphometry), PET imaging...

The three major software analysis package for neuroimaging: SPM (Statistical Parametric Mapping) and FSL (FMRIB Software Library) and AFNI (AFNI) provide implementations of massively univariate analyses. While there is a lot of commonalities across software packages, there are also software-specific outputs and specifities that can be of interest for the end-user.

This document describes the encoding of results of a massively univariate neuroimaging analysis using the NI-DM data model. The goal of this specification is to provide a unified representation of neuroimaging results across analysis software. When a piece of information is only available in a specific software, software-specific extensions are provided.

This is the third component of NI-DM and is referred to as NIDASH Results Data Model (NI-R DM).

Introduction

The NIDASH Results Data Model (NI-R DM) is a NI-DM compliant structured representation of the results of a massively univariate neuroimaging study.

The NI-R DM specification formally describes the encoding of the massively univariate neuroimaging results into a NI-DM representation. Extensions are provided to describe software-specific objects for SPM (Statistical Parametric Mapping) and FSL (FMRIB Software Library) analysis software.

Structure of this Document

Section 2 provides an overview of the NI-R data model, distinguishing a core set of types and relations, commonly found in provenance, from extended structures catering for more specific uses. It also introduces a modular organization of the data model in components.

Section 3 illustrates how the NI-R data model can be used to express the results of a neuroimaging study in SPM and FSL.

Section 4 provides the definitions (and examples) of NI-R concepts, structured according to three components.

Overview

This section introduces neuroimaging results concepts with informal explanations and illustrative examples. NI-R distinguishes software-agnostic structures, forming the essence of the results, from software-specific structures catering for more specific uses of results by different analysis software. Core and extended structures are respectively presented in Section 2.1 and Section 2.2. Furthermore, the PROV data model is organized according to components, which form thematic groupings of concepts (see Section 2.3). A provenance description is an instance of a provenance structure, whether core or extended, described below.

NI-R Core Structures

The NIDASH Results Data Model (NI-R DM) is concerned with the modelling of model fitting and inference in the context of massively univariate analyses. A typical example is the analysis of functional MRI data but studies involving other modalities (such as PET) and sequences (e.g. anatomical MRI through VBM) can also be modelled. The domain covered by NI-R data model is represented in Fig. 1.

The concepts found in the core of NI-Results are introduced in the rest of this section. They are summarized in Table 2, where they are categorized as type or relation. The first column lists concepts, the second column indicates whether a concept maps to a type or a relation, whereas the third column contains the corresponding name, as it appears in Figure 1. Names of relations have a verbal form in the past tense to express what happened in the past, as opposed to what may or will happen. In the core of PROV, all relations are binary.

Element 1: Model Fitting

ModelParametersEstimation

A ModelParametersEstimation ◊is a the process of estimating the parameters of a general linear model from the available data (sub-type of prov:Activity).

A ModelParametersEstimation ◊ has:
  • prov:label: an OPTIONAL label describing the ModelParametersEstimation.
activity(niiri:model_pe_id,
    [prov:type = 'nidm:ModelParametersEstimation',
    prov:label = "Model parameters estimation"])

Data

A Data ◊is a "A collection or single item of factual information, derived from measurement or research, from which conclusions may be drawn." (sub-type of prov:Entity).

A Data ◊ has:
  • prov:label: an OPTIONAL label describing the Data.
  • grandMeanScaling: an OPTIONAL binary flag defining whether the grand mean of the data was scaled.
  • targetIntensity: an OPTIONAL value to which the grand mean of the Data was scaled (applies only if grandMeanScaling is true).
  • medianIntensity: an OPTIONAL value corresponding to the median intensity of the Data.
entity(niiri:data_id,
    [prov:type = 'nidm:Data',
    prov:type = 'prov:Collection',
    prov:label = "Data" %% xsd:string,
    nidm:grandMeanScaling = "true" %%xsd:boolean,
    nidm:targetIntensity = "100" %% xsd:float,
    nidm:medianIntensity = "115.32" %% xsd:float])
